# interview 스킬

/ax-start 커맨드에서 호출된다.
사용자의 업무를 인터뷰하여 아래 5가지 필수 항목을 수집하는 것이 목표다.

## 수집 목표 항목

| 항목 | 설명 |
|---|---|
| 업무명 | 업무를 대표하는 짧은 이름 |
| 목적 | 이 업무를 왜 하는지 |
| 주기 | 얼마나 자주 하는지 (매일/매주/매월/비정기) |
| 애로사항 | 리소스가 많이 드는 부분, 반복 작업(특히 수기), 휴먼 에러 발생 지점 |
| 업무 단계 + 도구 | 단계별 흐름과 각 단계에서 사용하는 도구 |

모든 항목이 충분히 채워져야 인터뷰 완료로 처리한다.

---

## 자유 서술 방식 진행 원칙

사용자가 자유롭게 설명하도록 공간을 준다.
설명이 끝나면 빠진 항목만 자연스럽게 보완 질문한다.

**보완 질문 예시**

- 업무명/목적이 불명확할 때
  > "말씀해주신 업무를 한마디로 표현하면 어떻게 될까요?"

- 주기가 없을 때
  > "이 업무는 얼마나 자주 하시나요?"

- 애로사항이 없을 때
  > "이 업무에서 가장 번거롭거나 반복적으로 하시는 부분이 있나요?"

- 단계/도구가 불명확할 때
  > "업무 단계를 조금 더 구체적으로 말씀해주실 수 있나요? 각 단계에서 어떤 도구를 쓰시는지도 궁금해요."

**주의사항**
- 한 번에 여러 보완 질문을 던지지 않는다. 가장 중요한 빠진 항목부터 하나씩 묻는다.
- 사용자가 이미 말한 내용은 다시 묻지 않는다.
- 딱딱하게 묻지 않고 대화하듯 자연스럽게 이어간다.

---

## 구조화 질문 방식 진행 원칙

아래 4개 질문을 순서대로 하나씩 묻는다.

```
Q1. 어떤 업무를 담당하고 계신가요?
    이 업무를 왜 하는지 목적도 함께 알려주시면 좋아요.

Q2. 이 업무는 얼마나 자주 하시나요?
    (매일 / 매주 / 매월 / 비정기 등)

Q3. 이 업무에서 가장 힘든 점이 무엇인가요?
    리소스가 많이 드는 부분, 매번 반복되는 작업
    (특히 수기로 직접 하는 것들), 실수가 자주 나는 부분 등
    자유롭게 말씀해주세요.

Q4. 업무 단계를 순서대로 설명해주세요.
    각 단계에서 어떤 도구를 쓰는지도 함께 알려주시면 좋아요.
    (슬랙, 지라, 엑셀, 이메일, 구글 시트 등)
```

**답변이 불충분한 경우**
해당 질문에 한해 한 번만 추가 질문한다.

- Q1 답변에 목적이 없을 때
  > "이 업무를 하는 이유나 목표가 있다면 간단히 말씀해주세요."

- Q3 답변이 너무 짧거나 모르겠다고 할 때
  > "혹시 매번 비슷하게 반복하는 작업이 있나요? 예를 들어 같은 형식으로 정리하거나, 어딘가에 옮겨 적는 작업이요."

- Q4 단계 설명에 도구가 전혀 없을 때
  > "각 단계에서 엑셀, 슬랙, 이메일 등 사용하시는 도구가 있으면 알려주세요."

## 인터뷰 완료 판단 기준

아래 5가지가 모두 파악되면 완료로 처리한다.

- [ ] 업무명 (명시 또는 대화에서 추론 가능)
- [ ] 목적
- [ ] 주기
- [ ] 애로사항 (반복 작업 포함 여부 파악)
- [ ] 업무 단계 + 사용 도구 (최소 3단계 이상)

완료되면 아래 메시지를 출력한다.

---
인터뷰가 완료됐어요! ✅
/ax-publish 를 입력하면 산출물 3종을 만들고 위키에 게시해드릴게요.
