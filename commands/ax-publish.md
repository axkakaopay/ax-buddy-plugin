# /ax-publish

인터뷰에서 수집한 정보를 바탕으로 산출물 3종 초안을 생성하고,
사용자 확인 후 Confluence에 게시한다.

## 실행 조건

/ax-start 인터뷰가 완료된 상태여야 한다.
인터뷰 정보가 없으면 아래 메시지를 출력하고 종료한다.

> 먼저 /ax-start 로 업무 인터뷰를 진행해주세요.

## STEP 1. 산출물 3종 초안 생성

수집된 인터뷰 내용을 바탕으로 아래 3종을 작성해 채팅창에 보여준다.

### 1. 업무 프로세스 워크플로우
- 업무 단계를 순서대로 표현
- 각 단계 옆에 사용 도구를 이모지로 표시
  - 슬랙 💬 / 지라 📋 / Confluence 📄 / 이메일 📧
  - 엑셀·구글시트 📊 / PDF 📑 / 아지트 📢 / GWS 📅

### 2. 업무 정의서
| 항목 | 내용 |
|---|---|
| 업무명 | |
| 목적 | |
| 주기 | |
| Pain Point | |
| AX 전환 요약 | Claude가 인터뷰 내용 기반으로 핵심 자동화 포인트 요약 |

### 3. AI 활용 포인트
인터뷰에서 파악한 도구와 반복 작업을 기반으로 아래 2가지로 구분해 제안한다.

- MCP 연동형: 슬랙, 지라, Confluence, 아지트, GWS 등 연동 가능한 자동화 포인트
- 파일 처리형: 엑셀, CSV, PDF 등 파일 기반 자동화 포인트

## STEP 2. 작성자 및 업무명 확인

초안 출력 후 아래 메시지를 출력한다.

---
위키에 게시하기 전에 두 가지만 확인할게요.

1. 작성자 성함을 입력해주세요.
2. 업무명을 확인해주세요.
   (제안: {인터뷰 내용 기반으로 Claude가 업무명 제안})
   그대로 사용하거나 직접 수정해주세요.
---

## STEP 3. 게시 확인

작성자와 업무명이 확정되면 아래 메시지를 출력한다.

---
아래 내용으로 위키에 게시할까요?

- 페이지 이름: {업무명}_{작성자}
- 위치: 업무 프로세스 모음 하위

(네 / 수정 필요)
---

## STEP 4. Confluence 게시

사용자가 확인하면 wiki-publish 스킬을 사용해 페이지를 생성한다.

- 위치: 업무 프로세스 모음 (Confluence page ID: 1310721) 하위
- 페이지 이름: {업무명}_{작성자}

게시 완료 후 아래 메시지와 함께 링크를 출력한다.

---
위키 게시 완료! 🎉
{링크}
